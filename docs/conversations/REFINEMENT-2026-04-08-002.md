---

## 2. Root Cause Analysis

The `checkpoint_heartbeat.py` script exists for crash recovery but does NOT include conversation logging functionality. The session end (via `attempt_completion`) does not trigger any conversation capture.

### Existing Infrastructure

1. **`scripts/checkpoint_heartbeat.py`** — Writes heartbeat every 5 minutes, tracks Git state, enables crash recovery
2. **`memory-bank/hot-context/session-checkpoint.md`** — Stores session metadata
3. **`docs/conversations/`** — Destination for logged conversations
4. **`docs/conversations/README.md`** — Index of all conversations

### Missing Components

1. **Session end trigger** — No hook into `attempt_completion` to invoke logging
2. **Conversation capture** — No mechanism to capture the full conversation context
3. **Auto-start heartbeat** — No automatic start when mode begins

---

## 3. Proposed Solution

### Technical Approach: Extend checkpoint_heartbeat.py

Rather than creating a new script, extend the existing `checkpoint_heartbeat.py` with a `--log-conversation` flag that:
1. Captures session metadata from `activeContext.md` and `session-checkpoint.md`
2. Generates the conversation filename: `{YYYY-MM-DD}-{source}-{slug}.md`
3. Creates the conversation file with structured content
4. Updates `docs/conversations/README.md` with new entry

### Trigger Conditions

| Trigger | Action |
|---------|--------|
| Mode starts | Auto-start heartbeat (via VS Code task or shell profile) |
| Every 5 minutes | Heartbeat updates session-checkpoint.md |
| Session ends (`attempt_completion`) | Agent manually calls `checkpoint_heartbeat.py --log-conversation` |

### Conversation File Format

```markdown
---
session_id: s2026-04-08-architect-002
mode: architect
date: 2026-04-08
duration: ~45 minutes
files_modified: [...]
ideas_generated: [...]
---

# Conversation Summary

[Agent's summary of the conversation]

## Key Decisions

- Decision 1
- Decision 2

## Ideas Generated

- IDEA-NNN: Description

## Files Modified

- `path/to/file.md`
```
