## Discussion Summary

### Problem Statement Refinement

**Original problem:** Canonical docs still show "Draft" status when they should be "Frozen" after release.

**Refined problem:** The workbench lacks an explicit, enforceable status lifecycle for canonical documents. RULE 8 defines "Frozen" as read-only and "Draft" as modifiable, but does not specify:
1. When a document transitions from Draft → Frozen
2. Who is responsible for changing the status
3. How the transition should be verified (automated check)
4. What happens if a released document is still marked Draft

### Key Insight: SPEC vs ENFORCEMENT Split

During analysis, a critical insight emerged:

| Aspect | IDEA-014 | IDEA-015 |
|--------|----------|----------|
| **Role** | Policy/Specification | Enforcement/Implementation |
| **Delivers** | RULE 8 enhancement, status lifecycle definition | GitHub Actions release gate |
| **Validates** | Status transition rules documented | Status transition rules followed |

**Decision:** IDEA-014 is the SPEC, IDEA-015 is the ENFORCEMENT. They are complementary and should be accepted together.

### Historical Context

The v2.6 issue (DOC-1 showing Draft when it should be Frozen) was a symptom of missing enforcement, not missing policy. RULE 8.1 already states Frozen docs are read-only — it just wasn't being enforced. IDEA-015's release gate fixes this by automating the check.

### Status Lifecycle Defined

```
Draft → In Review → Frozen → (archived)
```

| Status | Transition Trigger | Enforced By |
|--------|-------------------|-------------|
| Draft | Default state during development | N/A |
| In Review | When PR is opened for canonical doc changes | Human review |
| Frozen | When release is tagged (vX.Y.0) | GitHub Actions release gate |

### Scope Decisions

**What's IN:**
- RULE 8 Enhancement: Add explicit status lifecycle definition
- RULE 8.1 Update: Clarify that Frozen means "immutable after release tag"
- Status Header Requirement: Every canonical doc MUST have a `status:` field in frontmatter
- DOC-*-CURRENT.md: Must reference the Frozen version immediately after release tag

**What's OUT:**
- Retroactively fixing v2.6 and earlier (already released)
- Modifying already-Frozen documents in docs/releases/vX.Y/ (forbidden by RULE 8.1)
- Real-time status tracking during development (too complex for v2.7)

### Dependencies

- IDEA-015 (Mandatory Release Coherence Audit) is **prerequisite** for AC-4
- IDEA-017 (Canonical Docs Cumulative) is informational — already implemented

### Complexity

- **Score:** 2/10
- **Reasoning:** This is primarily a RULE clarification and documentation update. The enforcement mechanism is already covered by IDEA-015's release gate.

## Decisions Made

1. **Split SPEC from ENFORCEMENT**: IDEA-014 delivers the policy specification; IDEA-015 delivers the enforcement mechanism
2. **Status Lifecycle Defined**: Draft → In Review → Frozen
3. **RULE 8 Enhancement Required**: Add explicit transition rules and status header mandate
4. **Complexity Reduced**: From 3/10 to 2/10 since enforcement is handled by IDEA-015
5. **Co-acceptance Recommended**: IDEA-014 and IDEA-015 should be marked [ACCEPTED] together since they depend on each other

## Final Requirements

### Acceptance Criteria

**AC-1: RULE 8 Enhanced**
- RULE 8.1 updated to define Draft → In Review → Frozen lifecycle
- Frozen means "immutable after release tag"
- Status header (`status:` in frontmatter) mandated for all canonical docs
- template/ updated with status header requirement

**AC-2: Status Transition Rules Defined**
- Draft → In Review: When canonical doc PR is opened
- In Review → Frozen: When release is tagged (vX.Y.0)
- No other transitions allowed

**AC-3: Pointer Consistency**
- DOC-1-CURRENT.md, DOC-2-CURRENT.md, DOC-4-CURRENT.md updated together
- DOC-3-CURRENT.md and DOC-5-CURRENT.md point to latest release-specific versions
- All DOC-*-CURRENT.md files show correct status

**AC-4: Release Gate Integration** (via IDEA-015)
- Release gate workflow checks all canonical docs for a release are Frozen
- If not Frozen when tagging → BLOCK
